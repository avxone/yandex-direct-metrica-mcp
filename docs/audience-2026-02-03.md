# Yandex Audience support (raw + HF + dashboards) — 2026-02-03

This MCP server supports **Yandex Audience API** in two layers:
- **Raw** (`audience.*`) — API-shaped payloads
- **HF** (`audience.hf.*`) — small inputs + concise results (+ `raw_refs`)

Write operations are **pro-only** and require standard guardrails (`apply=true` + write flags).

## Auth / env

By default, Audience uses the **main Yandex OAuth** token (same as Direct/Metrica).

Optional overrides (separate app/token):
- `YANDEX_AUDIENCE_CLIENT_ID`
- `YANDEX_AUDIENCE_CLIENT_SECRET`
- `YANDEX_AUDIENCE_REFRESH_TOKEN` (or `YANDEX_AUDIENCE_ACCESS_TOKEN`)

Enable/limits:
- `MCP_AUDIENCE_ENABLED=true|false` (default `true`)
- `MCP_AUDIENCE_RATE_LIMIT_RPS=0` (0 disables rate limiting)

## HF tools (public read-only)

- `audience.hf.find_segment`
- `audience.hf.get_segment_summary`
- `audience.hf.segment_health`
- `audience.hf.overlap_matrix`
- `audience.hf.segment_perf` (best effort; proxy via Direct targeting)
- `audience.hf.catalog`

Notes for `audience.hf.segment_perf`:
- Returns a daily series with **Direct** metrics (impressions/clicks/cost) when the segment can be mapped to Direct targets.
- Metrica segment-level filtering is not available; fields like `visits`/`goal_reaches` may be absent (status will be `partial`).

## Pro-only write tools

Raw:
- `audience.segments.create`
- `audience.segments.update`
- `audience.segments.delete`
- `audience.upload.start`
- `audience.upload.status`
- `audience.upload.errors`
- `audience.raw_call` (escape hatch; pro-only)

HF activation (Direct):
- `audience.hf.activation_plan` (preview-only)
- `audience.hf.apply_activation_plan` (executes when `apply=true`)

Current activation implementation:
- Creates a Direct `retargetinglist` referencing the Audience segment external id (best effort).
- Creates Direct `audiencetargets` for target adgroups (campaign targets are expanded to adgroups).
- Does **not** set bid modifiers (can be added later via Direct bid modifiers).

## Dashboard integration (Option 1)

`dashboard.generate_option1` supports:
- `include_audience=true` — adds `data.audience` block and renders an “Audience” card in HTML.

## BI Option 2 (datasets + incremental sync)

Tools:
- `dashboard.schema`
- `dashboard.dataset.audience_segments`
- `dashboard.dataset.audience_overlap`
- `dashboard.dataset.audience_segment_perf_daily`
- `dashboard.sync.start`
- `dashboard.sync.next`

`dashboard.sync.*` returns **NDJSON** pages and uses an opaque `cursor` (base64url JSON).
