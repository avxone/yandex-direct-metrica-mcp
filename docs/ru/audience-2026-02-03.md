# Поддержка Yandex Audience (raw + HF + dashboards) — 2026-02-03

Этот MCP‑сервер поддерживает **Yandex Audience API** в двух слоях:
- **Raw** (`audience.*`) — ответы близко к форме API
- **HF** (`audience.hf.*`) — компактный вход + короткий результат (+ `raw_refs`)

Write‑операции — **только PRO** и требуют стандартные guardrails (`apply=true` + write flags).

## Auth / env

По умолчанию Audience использует основной OAuth‑токен Яндекса (тот же, что и для Direct/Метрики).

Опциональные overrides (отдельное приложение/токен):
- `YANDEX_AUDIENCE_CLIENT_ID`
- `YANDEX_AUDIENCE_CLIENT_SECRET`
- `YANDEX_AUDIENCE_REFRESH_TOKEN` (или `YANDEX_AUDIENCE_ACCESS_TOKEN`)

Включение/лимиты:
- `MCP_AUDIENCE_ENABLED=true|false` (по умолчанию `true`)
- `MCP_AUDIENCE_RATE_LIMIT_RPS=0` (0 отключает rate limit)

## Raw tools (public read‑only)

- `audience.user_info`
- `audience.segments.list`
- `audience.segments.get`
- `audience.segments.stats` (best effort)
- `audience.segments.overlap` (best effort)
- `audience.pixels.list`
- `audience.pixels.get`
- `audience.lookalikes.list` (best effort)
- `audience.lookalikes.get` (best effort)

## HF tools (public read‑only)

- `audience.hf.find_segment`
- `audience.hf.get_segment_summary`
- `audience.hf.segment_health`
- `audience.hf.overlap_matrix`
- `audience.hf.segment_perf` (best effort; proxy через таргетинг Direct)
- `audience.hf.catalog`

Примечания к `audience.hf.segment_perf`:
- Возвращает дневной ряд с метриками **Direct** (показы/клики/расход), если сегмент удаётся сопоставить с таргетингом Direct.
- Фильтрации Метрики на уровне сегмента нет; поля `visits`/`goal_reaches` могут отсутствовать (статус будет `partial`).

## PRO write tools

Raw:
- `audience.segments.create`
- `audience.segments.update`
- `audience.segments.delete`
- `audience.upload.start`
- `audience.upload.status`
- `audience.upload.errors`
- `audience.raw_call` (escape hatch; pro‑only)

HF activation (Direct):
- `audience.hf.activation_plan` (только preview)
- `audience.hf.apply_activation_plan` (выполняет при `apply=true`)

Текущая реализация activation:
- Создаёт Direct `retargetinglist`, который ссылается на external id сегмента Audience (best effort).
- Создаёт Direct `audiencetargets` для adgroups (campaign targets разворачиваются до adgroups).
- **Не** выставляет bid modifiers (можно добавить позже через инструменты модификаторов ставок Direct).

## Интеграция в Dashboard (Option 1)

`dashboard.generate_option1` поддерживает:
- `include_audience=true` — добавляет блок `data.audience` и рендерит карточку “Audience” в HTML.

## BI Option 2 (датасеты + инкрементальный sync)

Инструменты (PRO plug-in):
- `dashboard.schema`
- `dashboard.dataset.audience_segments`
- `dashboard.dataset.audience_overlap`
- `dashboard.dataset.audience_segment_perf_daily`
- `dashboard.sync.start`
- `dashboard.sync.next`

`dashboard.sync.*` возвращает страницы **NDJSON** и использует opaque `cursor` (base64url JSON).
