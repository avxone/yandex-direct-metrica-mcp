# v2.0.12

Date: 2026-06-19

## Summary

This release hardens the Yandex Search API Wordstat integration, adds a first real Symphony-oriented delivery pipeline around the repository, and includes the current PRO HTML dashboard work that was already present in the local tree and validated before release.

## Highlights

- Wordstat:
  - fixed `wordstat.regions` mapping to `region: REGION_*`;
  - merged provider `associations` into HF/dashboard candidate generation;
  - validated `wordstat.dynamics` month/week date constraints with actionable failures;
  - changed `wordstat.user_info` into a live `getRegionsTree` access check;
  - improved Search API setup and error hints.

- Automation:
  - added Linear intake CLI and templates;
  - documented the Symphony state machine, roles, and release gates;
  - added changed-line lint, release guard, and live validation scripts;
  - added GitHub Actions workflows for live validation and GitHub Releases.

- Dashboard PRO HTML:
  - included the current `dashboard.generate_pro_html` implementation and regression coverage already present in the validated working tree.

## Validation

Release validation completed with:

- `python -m compileall -q src/mcp_yandex_ad`
- `pytest -q`
- changed-line lint via `python scripts/agent_lint.py`
- real live checks against configured Yandex credentials:
  - Direct
  - Metrica
  - Search API Wordstat
