# v2.0.10

Date: 2026-05-08

## Changes

- MCP review fixes:
  - unique generated Direct `ReportName` values across low-level and HF report helpers
  - valid `direct.hf.report_keywords` `CUSTOM_REPORT` field set
  - actionable `direct.report` validation for `Keyword` vs `Criterion`
  - auto-pagination for `metrica.hf.report_landing_pages`, `metrica.hf.report_utm_campaigns`, `metrica.hf.report_geo`, and `metrica.hf.report_devices`
  - Wordstat batch fallback for `wordstat.top_requests`
  - special/no-structure campaign diagnostics in `direct.hf.get_campaign_summary`
  - read-only Direct agency login override support
  - HF discovery pagination and filter corrections
- Contracts: `dashboard.generate_option1` is no longer annotated as read-only when `output_dir` writes local artifacts.
- Docs: added a focused contract update note for `Marketing2025` consumers and detailed cross-repo handoff documents.

## Consumer Notes

- `direct.hf.get_campaign_summary` may return:
  - warning code `campaign_type_special_no_structure`
  - result fields `campaign_type`, `campaign_type_hint`, `counts_applicable`, `performance_signal`
- `direct.list_campaigns` may substitute special campaign candidates when an ID-based lookup returns no rows and live performance indicates a special/no-structure campaign.
- `metrica.hf.report_*` stats tools now fetch full result sets by default; if you pass an explicit `limit`, expect warning code `metrica_rows_truncated` when rows are cut off.
- `wordstat.top_requests` may return `fallback.mode="single_phrase_loop"` and `topRequestsByPhrase` when batch mode fails upstream.
- `dashboard.generate_option1` with `output_dir` writes local HTML/JSON files and should be treated as side-effecting with respect to the filesystem.

## Notes

- Public vs Pro tool availability remains unchanged.
- This release prepares the backend side for the next shared `Marketing2025` replay validation.
