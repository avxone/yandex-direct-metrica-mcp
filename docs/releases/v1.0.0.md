# Release v1.0.0 (Public + Pro artifacts)

This is the first **1.x** release of `yandex-direct-metrica-mcp`.

## Highlights

- Full contour: **Direct + Metrica + Wordstat + Audience** (raw + HF) plus join tools and dashboards.
- Public contract (read-only) is stabilized via `tools/list` snapshot:
  - `tests/snapshots/public_tools_v1.json`
- Public edition is **safe-by-default**:
  - public Docker image forces read-only even if runtime env vars are misconfigured.
- Separate artifacts:
  - Public image: `ghcr.io/<OWNER>/yandex-direct-metrica-mcp:v1.0.0` and `:latest`
  - Pro image: `ghcr.io/<OWNER>/yandex-direct-metrica-mcp-pro:v1.0.0` (intended for paid subscribers; keep GHCR package private)

## Public read-only definition (Contract A)

Read-only means: no changes to managed entities in Direct/Metrica/Audience.

Allowed provider-side side effects (still treated as read-only):
- Wordstat report-like requests
- Metrica Logs API export jobs (`metrica.logs_export`) for analysis/joins

## PRO additions (not part of public contract)

- BI Option 2 (datasets + incremental sync): `dashboard.schema`, `dashboard.dataset.*`, `dashboard.sync.*`
- Direct write via plan/apply: `direct.hf.plan_changes` → `direct.hf.apply_plan`
- Metrica goals CRUD (raw + HF apply-guarded): `metrica.goals.*`, `metrica.hf.*`
